## 2-2. 문제 상황 및 해결 과정

- **Situation**
    - 피험자의 '수면 시작 시간'이나 '기상 시간'을 직접적으로 알려주는 **명시적인 라벨** X
    - 10분, 1분 단위로 수집되는 조도(Light), 충전(AC), 화면(Screen) 센서 데이터는 기기 미착용이나 일시적 조작 등으로 인해 **노이즈/미측정 구간** 많았음.
    - 피험자마다 심한 개인차 존재
- **Task - Action**
    - **T:** 어떻게 수면 시작시간 / 기상시간같은 변수들을 추정할 것인가?
        - **A:** 충전상태 / 화면 켜짐 상태 데이터, 조도 데이터의 교집합을 수면 상태로 정의
    - **T:** 미측정된 구간을 어떻게 처리할 것인지? (해당 데이터를 제거할 것인지, 보간할 것인지 등)
        - **A:** 미측정 구간이 연속적(하루 단위로 데이터가 측정되지 않은 경우 등): 피쳐로 해당 데이터셋의 날짜별 Z-score를 데이터의 신뢰도로 사용 → 모델이 해당 신뢰도까지 함께 학습할 수 있도록 함.
        - **A:** 미측정 구간이 간헐적(데이터가 1~2단위 정도 측정되지 않은 경우 등): 10분/30분 단위 롤링 합계 및 이동평균 적용
    - **T:** 피험자간 개인차를 어떻게 고려할 수 있을지?
        - **A:** 데이터셋에서 피험자별 유형을 구분하는 피쳐를 추가
             - 22~06시 사이의 조도의 분산이 작음 → 규칙적인 수면 패턴 = 1 분류.
- **Result**
- 개별 센서끼리 서로 상호 보완한 수면/활동 프록시 변수 도출
- 추후 모델 앙상블 과정에서 개인별 데이터의 일관성을 측정하여 가중치를 부여하는 기반 데이터로 활용.

- 시간대별 사용시간 TOP5 어플 카테고리 (app_evening_Rank1_app_category …)
- 시간대별 사용시간 TOP5 어플 사용시간 (app_evening_Rank1_app_total_time_sec …)
- 시간대별 사용시간 TOP5 소음 이름 (ambience_day_Rank1_sound …)
- 시간대별 사용시간 TOP5 소음 카테고리 (ambience_day_Rank1_category …)
- 시간대별 사용시간 TOP5 소음 강도 (ambience_day_Rank1_intensity …)

### tsfresh 라이브러리

- 조도(mLight, wLight), 심박수(wHr) 등의 시계열 데이터에서 분산, 피크 횟수, 자기상관성 등의 통계적 피처를 대량 추출
- 모델 특성상 고차원의 인풋이 들어갈 수 없음
   → 시계열 데이터 압축 과정에서 손실 발생
      → 시계열 정보를 반영한 피쳐를 생성

## 4. 모델링 및 앙상블

### 4.1. 모델 앙상블 근거

- 데이터 수가 적음 → 과적합 방지를 최우선!
- **CatBoost**
    - 대칭 트리 구조, 과적합 방지에 탁월.
    - mUsageStats(앱 사용내역), mAmbience(소음 종류) 등 문자열 카테고리 데이터를 별도의 전처리 없이 학습 가능
- **XGBoost**
    - early_stopping → 과적합 피할 수 있음
    - 범주형 변수는 라벨 인코딩 전처리 필요 → 범주형은 XGBoost 모델 사용X
- **LightGBM**
    - Leaf-wise 기반 모델: 과적합에 취약
    - 파라미터 튜닝 통해 규제 강하게 할 필요 있음

### 4.2. 모델 검증 전략 (CV)

- **Subject-Hole CV (Purged Group Time-Series CV):**
    - 기존 Stratified K-Fold 교차검증에서 보완
    - 기존 K-fold CV: 전체 데이터 중 무작위로 분할
    ⇒ 시간순 정렬 후 데이터 블록 만듦
    ⇒ 사용자별로 검증 데이터 분할 (Subject)
    - 제출 포맷과 유사하게 테스트셋 분할
    ⇒ 데이터 중간중간 의도적으로 공백 만듦 (Hole)

### 4.3. 파라미터 튜닝: optuna

- optuna: 베이지안 최적화 기반의 알고리즘 활용한 하이퍼파라미터 최적화
- 일반적인 grid search 방법보다 효율적
